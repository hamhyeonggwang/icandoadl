# 구현 계획 — 3D ADL 훈련과제 저작·실행 도구 v0.1
날짜: 2026-07-12 · 전제: Gate 1 스파이크 셀프 테스트 전 항목 통과 (아동 대상 정식 Gate 1은 별도 진행)
상위 문서: problem-brief_adl-webcam-trainer_v0.1.md — 동결 결정(A1·A1′·A5′·A6·A7)·Out/Exit·C2 전부 계승

## 목표
치료사가 코드 없이 저작한 Session(JSON)을 아동이 웹캠 PC 브라우저에서 링크로 실행.
러너는 **3D 항행(3인칭) ↔ 거울 모드 조작(스테이션)** 페이즈 교대. 상호작용은 화면 평면 2.5D, 3D는 렌더링 전용(브리프 동결).

## 게임의 인지 목적 (2026-07-12 명문화)
ADL 수행의 신체 훈련을 넘어, 아동이 **상황을 인지하고 실행기능을 경험**하는 것이 핵심 목적:
- **주의·집중**: 게임 흐름을 놓치지 않고 자기 차례·신호를 주시
- **작업기억**: 이전 단계(선택·순서)를 기억해 다음 행동 계획
- **충동 억제**: 즉각 행동 욕구 조절 — 빨간불 기다리기, dwell 유지, 예산 초과 참기
- **결과 예측·인과 추론**: "이 행동을 하면 다음에 무슨 일이?" (신호·예산·순서)
→ 각 과제에 인지기능 배지를 달아 치료사가 목표에 맞게 과제를 선택하도록 함.

## 아키텍처
```
app/
  runner.html / editor.html      ← 빌드 없는 정적 페이지 (퐁퐁 단일파일 철학 계승, 모듈만 분리)
  js/
    params.js        스파이크 PARAMS 계승 (전부 초기 추정 — 캘리브레이션·현장값으로 갱신)
    session.js       Session/Station/Segment 스키마 + 링크 인코딩(base64url) + 데모 세션
    library-meta.js  내장 일상사물 라이브러리 메타 (에디터·러너 공용, three 비의존)
    library-mesh.js  라이브러리 3D 프로시저럴 메시 (three 의존, 러너 전용)
    input-hand.js    HandDriver — MediaPipe Hand+Pose, 캘리브레이션, 분류·스트로크 검출 (스파이크 이식)
    input-mouse.js   MouseDriver — 개발·폴백용 (마우스=손, F/J=교대 스트로크, A/D=기울기)
    runner-app.js    러너 엔진: 페이즈 FSM, 항행 씬, 스테이션 씬, R1/R2 쥐기 FSM, assist
    editor-app.js    에디터: 플로우 편집, 사물 배치, 단계조절, JSON 저장/불러오기, 실행 링크
```
- **입력 드라이버 추상화**: 브리프 Exit 조항("Gate 1 실패 시 마우스·터치 폴백") 선반영. 드라이버 출력 계약:
  `{ hands:{L,R: {present, openness, cls, pos(표시좌표계), vy}}, lean, strokeEvent }`
- **좌표 규약**: 전 계층 표시좌표(거울 반영 후) 정규화 0~1. HandDriver가 `x → 1−x` 변환 담당.
- **C2 불가침**: 영상 프레임 저장·전송 없음. 세션 JSON에도 영상·아동정보 없음.

## 데이터 스키마 (Session v1)
```
Session { version, title, flow: (Segment|Station)[] }
Segment { type:'segment', level:0|1|2, length:'short'|'medium'|'long', theme }
Station { type:'station', title, instruction,
          items:[{lib, pos:[x,y], scale, hand:'any'|'L'|'R'}],   ← 잡기 대상
          target:{lib, pos:[x,y], scale, zoneRadius},            ← 드롭존 (v1: 스테이션당 1개)
          grading:{graspRadius, tReleaseMs, assistTimeoutS} }
```
v1 단순화: 스테이션당 타깃 1개, 모든 아이템 → 동일 타깃 ("가방에 필통·책 넣기"). 다중 타깃 페어링은 v1.1.

## M5.5 — 마을 콘텐츠 확장 (2026-07-12 구현·검증)
- **crossing 플로우 타입**: 2차선 도로 + 보행 신호등 + 차량 4대 + 얼룩말무늬 + 건너편 마트 파사드.
  - 훈련 규칙: 빨간불 = 연석에서 전진 잠김(실패 아님, 코칭 문구) → 초록불에만 출발(깜빡임엔 다음 신호 대기) → Level 2는 **손 번쩍 들기(OPEN + 화면 상단)**가 전진 게이트(어깨 거상 훈련 결합) → 건너는 중 신호 변경 시 차량이 정지선 대기(실패호 제거).
  - 신호 음향(초록 멜로디·깜빡임 틱) = 실제 음향신호기 유사, 시각+청각 단서.
  - 단계조절: Level 0/1/2, 빨간불·초록불 길이 조절.
- **장소(place) 시스템**: 스테이션에 마을 공간 11종(아이방·주방·화장실·세면대·현관/신발장·문구점·마트·카페·식당·학교) — 장소 칩 + 색 데코 + (마우스 모드) 배경 틴트.
- **항행 테마 6종**: 집 복도·엘리베이터·길거리·공원·마트 통로·학교 복도.
- **라이브러리 34종**: 과일(바나나·귤·포도·딸기), 빵·주스, 신발·옷, 연필·공책, 드롭존(장바구니·신발장·옷장·세면대·접시·책상) 등 추가.
- **데모 세션 "우리 마을 하루"**: 아이방(옷 정리)→세면대(양치)→주방(아침)→현관(신발)→엘리베이터→길거리→🚦횡단보도→마트(과일 담기)→문구점→카페→학교 복도→교실(책상 정리) — 마을 한 블록 완성.
- 도로는 crossing이, 엘리베이터는 항행 테마가 담당 (스테이션은 조작 과제가 있는 공간만).

## M5.6 — 치료 깊이 확장 (2026-07-12 구현·검증)
- **세션 리포트 (A2 승격 결정)**: 자동 기록은 치료사 동석 관찰의 *보조*로 승격. C2 준수 — 파생 수치만, 온디바이스, 완료 화면 요약 + JSON/CSV 로컬 다운로드. 지표: 스테이션(잡기 시도/빈손, 존 밖 놓기, 방해자극 시도, 추적소실, 도움 단계, 소요시간), 횡단보도(신호대기·건너기 시간, 대기 중 출발시도, 손내림 횟수), 이동(스트로크 수).
- **변별 과제(distractor)**: "과일**만** 담아요" — 방해 자극을 존에 넣으면 부드럽게 되돌아옴(실패 아님, 지표만 기록). assist 부유 이동에서 제외. 에디터 체크박스 + 🙅 뱃지.
- **양손 과제(hand:'both')**: 큰 사물은 두 주먹이 모두 반경 안이어야 들림, 두 손 중점을 따라 이동, 한 손 놓기/추적소실 시 전체 해제. **R1은 양손 과제에 한해 완화**(양손 동시 조건이 이미 상위 난도) — 결정 근거 기록. 브리프 목표 '양측 상호교대 협응'을 조작 과제로 직결.
- **별 보상**: assist 단계를 척도로 재활용 — 무도움 ⭐⭐⭐ / 1단계 ⭐⭐ / 2단계 ⭐. 완료 배너·리포트·세션 총계 표시. 완료 후 assist 동결(별-도움 불일치 버그 수정).
- **다음 후보 (보류)**: 횡단보도 좌우 살피기(고개 회전) — Pose yaw 신뢰성 미검증, A8류 스파이크 후 결정.

## M5.7 — IADL 시뮬레이션 (2026-07-12 구현·검증)
**아키텍처 결정 (v1.1)**: 버스 노선 확인·키오스크 주문·셀프 계산대·길찾기 앱의 공통 원형은
"화면 앞에서 고르기". 페이즈 폭발 대신 스테이션 `kind: 'move'|'select'` 하나로 통합 (단독 유지보수 제약).
- **고르기(select) 스테이션**: 선택 카드 패널(캔버스 텍스처 — 외부 자산 없음, C2 유지). 순차 스텝 스크립트
  (예: 키오스크 = 메뉴 선택 → 결제 버튼). 선택 동작 = **OPEN 손 dwell(머무르기)** — 상지 도달+유지 훈련.
  오답 = 흔들림+기록(실패 없음). assist: 타임아웃 → 정답 강조 → 오답 흐리게. 단계조절: dwellMs 400~2000.
- **예산 장보기(셀프 계산대)**: move 스테이션 옵션 — `budget`(원) + 사물 `price`(가격표 스프라이트) +
  `requiredCount`(N개만 골라 담기). 예산 초과 담기 = 되돌아옴(실패 없음, 지표 기록).
  **담은 것 빼기 허용** — 검증 중 발견한 조합 데드락(비싼 것 먼저 담으면 완료 불가)의 해법이자
  실제 계산대의 '빼기' = 조합 재선택(집행기능) 훈련.
- **데모 14항목**: 버스 정류장(3번 버스 고르기→앞문 타기), 마트 예산 장보기(5,000원·2가지),
  카페 키오스크(딸기주스→결제), 길찾기 앱(방향 고르기) 추가. 장소 busstop·street 추가.
- 리포트 지표 추가: 오선택(wrongSelects)·예산초과 시도(budgetOvers)·스텝 수.
- 에디터: "+ 고르기" 버튼, 스텝/선택지 편집기(정답 체크), dwell 슬라이더, 예산·가격 필드, 카드 미리보기.

## M5.8 — 현장(웹캠) 피드백 재정비 (2026-07-12)
1. **횡단보도 손들기 폐기** — 원인 규명: 스트로크(양손 주먹)와 손들기(편 손 위로)가 **동작 모순**이라
   Level 2에서 이동 불가. 게이트 삭제, Level 0 자동/1 스트로크로 단순화. "상황에 맞는 이동" 초점.
   합성 입력 검증: 빨간불 출발시도 17회 차단 → 초록불 스트로크 22회로 완주.
2. **세면대 메시 재설계** — 수전이 z축 뒤에 있어 정사영 뷰에서 분리돼 보이던 문제 →
   받침대-세면볼-수전기둥-스파우트 일체형(같은 평면).
3. **신발 과제 방향 반전** — 외출 서사에서 신발은 신발장에 '넣는' 게 아니라 '꺼내 신는' 것 →
   신발 → 발판(footmat 👣, 신규 사물) 과제로 교체.
4. **엘리베이터 = 버튼 스테이션** — 전진 항행(실외처럼 보임) 폐기 → "1층 버튼을 눌러요" 고르기
   스테이션(장소 🛗 신규). 엘리베이터 항행 테마는 에디터에 잔존(선택 가능).
5. **데모 v3 "우리 마을 하루" (15항목)** — 일상 서사 순서로 재배열: 옷 갈아입기 → 세면대 → 아침 →
   신발 꺼내 신기 → 🛗 1층 버튼 → 골목 걷기 → 🚏 버스 → 큰길 → 🚦 횡단보도(L1) → 🛒 예산 장보기 →
   ✏️ 문구점 → ☕ 키오스크 → 🛣️ 길찾기 → 학교 복도 → 교실(양손 책).
- 난이도 호: 단순 옮기기(아침) → 선택 1스텝 → 신호 준수 → 예산·변별 → 다단계 선택 → 양손 협응(종착).

- **두 진입점**: 시작 화면에 "🌟 전체 하루 시작"(순차) / "🎯 과제 골라 연습"(선택).

  + **인지기능 배지**(주의·작업기억·충동억제·인과추론 등, 실행기능 태그 우선 slice(0,3)).
- **개별 실행 → 메뉴 복귀**: 한 과제를 끝내면 별점 토스트와 함께 메뉴로 돌아옴(연습한 카드는 dim+별점).
  여러 과제를 필요에 따라 골라 반복 연습 가능 — 전체 하루를 다 할 필요 없음.
- **캘리브레이션 분리**: 웹캠 모드에서 "🖐️ 준비운동" 카드로 독립. 다른 과제 첫 선택 시 미완료면 자동 선행,
  이후 과제는 스킵. 마우스 모드는 카드 숨김(불필요). `ensureDriver()`/`runCalibration()`로 리팩터링.
- 검증: 마우스 모드에서 메뉴→개별 실행(횡단보도·물건옮기기)→복귀→별점, 뒤로→시작, 전체 하루 순차 실행 확인.
