# Part A — 공통 (전 페이지 공유)

## 1. 개요
- **제품 목적**: 아동이 웹캠 앞에서 손·몸을 움직여 일상생활(ADL)을 수행하며, 신체 기능과 실행기능을 훈련한다.
- **해결하는 문제**: 치료사가 아동 목표에 맞춘 디지털 ADL 과제를 코드 없이 만들 수단이 없다. 기존 도구는 목표 정합성이 낮고 과제가 파편적·미완성이다.
- **핵심 가치**: ① 치료 목표별로 구분된 과제, ② 웹캠만으로 신체+인지 동시 훈련, ③ 영상 온디바이스(전송 없음).
- **재정비 방향(v0.1)**: 넓이(많은 과제) → 깊이(각 과제를 사용 가능 수준으로) + 순차 개방.

## 2. 대상 사용자 (3 페르소나)
| 페르소나 | 맥락 | 핵심 니즈 |
|---|---|---|
| **아동 (러너·최종 사용자)** | 치료실 웹캠 PC, 치료사 동석 | 실패 종결 없는 놀이, 명확한 피드백, 신체 움직임=조작 |
| **치료사 (저작자·운영자)** | 세션 전 저작, 세션 중 관찰 | 목표별 과제 선택, 코드 없는 저작, 단계조절, 수행 관찰 보조 |
| **병원 IT·개인정보 담당** | 도입 심사 | 영상 온디바이스, 저장·전송 없음 확인 |

## 3. 목표 & 성공 지표
| 목표 | 지표 | 목표값 |
|---|---|---|
| 과제별 "사용 가능" 도달 | 웹캠 실기 통과 기준(각 페이지 §수용 기준) 충족 | 개방된 과제 100% |
| 아동 완주 (개방된 과제) | 실패 종결 횟수 | 0 (assist 허용) |
| 치료사 저작성(Gate 2) | 미참여 치료사 무도움 저작 시간 | ≤ 30분 |
| 결합 성능 | Hand+Pose 동시 구동 FPS | ≥ 20fps |

## 4. 기술 아키텍처 (실제 구조 — 기본 스택 미적용)
- **순수 정적**: 빌드 없음. HTML + ES Module JS. three.js·MediaPipe tasks-vision는 CDN importmap.
- **백엔드/DB/인증 없음** — 의도된 설계(개인정보 경계). 서버가 없으므로 전송할 곳도 없다.
- 입력 드라이버 추상화: `HandDriver`(MediaPipe Hand+Pose) / `MouseDriver`(개발·폴백) — 동일 출력 계약 `{hands:{L,R}, lean, roll, strokeEvent}`.
- 세션 데이터: JSON(+GLB data URL). 링크는 base64url 해시(서버 불요). 러너·에디터 분리.
- 배포: 정적 호스팅(Vercel). `serve.mjs`는 로컬 전용.

## 5. 비기능 요구사항
- **성능**: 결합 20fps↑, pose는 3프레임당 1회 추론. 저사양 PC 고려.
- **접근성(아동)**: 큰 타깃·명확한 색 대비·즉각 피드백·읽기 부담 최소(이모지+짧은 문구). 소리 단서 병행.
- **반응형**: 웹캠 PC(16:9) 기준. 세로/모바일은 범위 밖(브리프 Out).
- **오프라인 내성**: 최초 로드 후 CDN 캐시. (⚠️ 완전 오프라인은 §결정 필요)

## 6. 데이터·개인정보 (불가침 — C2)
- 웹캠 영상 프레임: **저장·전송 절대 금지**. 렌더링·추론에만 메모리에서 사용.
- 파생 수치(개방도·손목 속도·어깨 변위)만 세션 리포트로 소비 → 로컬 다운로드(JSON/CSV)만.
- 세션 JSON·리포트에 아동 식별정보 없음. 클라우드 연동 없음.

## 7. 순차 개방(Gating) 시스템 — 전 러너 공통 규칙
- 스테이지 선택 메뉴는 **모든 과제를 표시**하되, "사용 가능" 미달 과제는 **🔒 잠금**(실행 차단, 안내 문구).
- 개방 상태는 앱 상수(개방 목록)로 관리 — 과제 하나가 §수용 기준을 통과하면 목록에 추가.
- "전체 하루" 시작도 잠긴 과제를 건너뛰거나(또는 잠금 시 비활성) — ⚠️ §결정 필요.
- 개방 순서(제안): 캘리브레이션 → 물건 옮기기 → 걷기 → 횡단보도 → 고르기 → 확장(변별·양손·예산).

# Part B — 페이지별 PRD

> 각 블록 공통 항목: 목적 / 치료 목표 / 현재 상태 / P0 요구사항(사용 가능) / 화면 요소 / 상호작용·상태 / 수용 기준(=개방 조건) / 잠금·개방 / ⚠️ 결정 필요

## P1. 랜딩 페이지
- **목적**: 방문자를 러너(아동) 또는 에디터(치료사)로 분기.
- **치료 목표**: 없음(진입).
- **현재 상태**: 러너/에디터 2카드 + 인지영역 안내. 배포 확인됨.
- **P0 요구사항**: 두 진입 명확, 온디바이스 안내 1줄, 웹캠 없이 둘러보기(`?input=mouse`) 안내.
- **화면 요소**: 제목, 소개 1문장, [아동 러너]/[치료사 에디터] 카드, 훈련 인지영역 배지.
- **상호작용**: 카드 클릭 → 해당 페이지.
- **수용 기준**: 데스크톱/모바일에서 레이아웃 정상, 두 링크 정확 이동, 초회 방문자가 어디로 갈지 3초 내 판단.
- **잠금·개방**: 해당 없음.
- **⚠️ 결정 필요**: 랜딩에 "치료사용/아동용" 이상의 소개(효과·근거)를 둘지. OTHub.kr 임베드 시 랜딩 생략 여부.

## P2. 러너 홈 (시작 화면)
- **목적**: 러너 실행 모드 선택 — 전체 하루 / 과제 골라 연습.
- **치료 목표**: 없음(진입). 단, **치료사가 목표에 맞는 진입을 고르는 지점**.
- **현재 상태**: 세션 제목·구성 요약, [전체 하루 시작]/[과제 골라 연습], 세션 파일 열기.
- **P0 요구사항**: 웹캠/마우스 모드 자동 판별, 모델 로딩 상태 표시, 카메라 실패 시 명확한 안내+마우스 폴백 경로.
- **화면 요소**: 세션 제목, 구성 요약(스테이션/이동/횡단보도 수), 2개 진입 버튼, 파일 열기.
- **상호작용**: 전체 하루 → 캘리브(웹캠·미완 시) → 순차 실행. 골라 연습 → 과제 메뉴.
- **수용 기준**: 카메라 권한 거부·미연결 시에도 앱이 죽지 않고 안내, 마우스 모드 진입 가능.
- **잠금·개방**: "전체 하루"는 개방된 과제만 포함할지 ⚠️(§7).
- **⚠️ 결정 필요**: 개방 과제가 1개뿐일 때 "전체 하루" 버튼을 숨길지/유지할지.

## P3. 과제 선택 메뉴 (핵심 — 순차 개방)
- **목적**: 치료 목표별로 개별 과제를 골라 반복 연습. **미완성 과제는 잠금 노출**.
- **치료 목표**: 페이지 자체는 없음. 각 카드가 목표(인지·신체 배지)를 드러내 **치료사의 목표 기반 선택**을 지원.
- **현재 상태**: flow 항목을 카드로 나열(번호·이모지·제목·유형·인지 배지), 캘리브 카드(웹캠), 개별 실행 후 복귀+별점.
- **P0 요구사항**:
  - **잠금 표시**: 미개방 과제 카드에 🔒 + "준비 중" 문구, 클릭 시 실행 차단(안내 토스트).
  - 개방 과제만 진입 가능. 개방 여부는 앱 개방 목록 기준.
  - 각 카드에 **치료 목표 배지**(신체+인지) 유지 — 치료사가 목표로 고를 수 있게.
- **화면 요소**: 뒤로, 제목, 안내, (준비운동 카드), 과제 카드 그리드[번호·이모지·제목·유형·목표 배지·🔒 또는 별점], 완료 토스트.
- **상호작용**: 개방 카드 → (필요 시 캘리브 선행) → 해당 과제 1개 실행 → 메뉴 복귀(별점·dim). 잠금 카드 → "아직 준비 중이에요" 안내.
- **수용 기준**: 개방/잠금 상태가 카드에 명확, 잠금 클릭이 오작동 없이 차단, 개별 실행→복귀 흐름 안정.
- **잠금·개방**: 본 페이지가 개방 시스템의 UI. 개방 목록 변경만으로 카드 상태 갱신.
- **⚠️ 결정 필요**: 목표(영역)별 필터/그룹핑을 넣을지(예: 상지/인지/안전 탭). 카드 정렬(개방 우선 vs 서사 순).

## P4. 캘리브레이션
- **목적**: 아동별 손 개방–폐쇄 범위와 체간 중립을 측정해 이후 손 과제의 기준을 잡음.
- **치료 목표**: 치료 과제 아님. 부수적으로 능동적 손 폄·쥠 최대 수행 유도(워밍업), 자세 정렬 인식.
- **현재 상태**: 15초 3단계(펴기 5s→쥐기 5s→중립 5s), 카운트다운, 폴백 범위.
- **P0 요구사항(사용 가능)**:
  - 아동이 각 단계에서 **무엇을 할지 즉시 이해**(그림/시연 큐, 문구 최소).
  - 손 미검출 시 진행 대신 "손을 화면에 보여주세요" 유도, 실패 시 재시도.
  - 캘리브 품질 자가 점검: OPEN/FIST 분포가 히스테리시스 대역을 침범하면 재캘리 권고.
- **화면 요소**: 단계 제목, 안내(가급적 시각 큐), 카운트다운, (권장) 손 검출 표시.
- **상호작용**: 자동 진행. 준비운동 카드/재캘리로 재실행.
- **수용 기준**: 비정형 손자세 아동 포함, 캘리브 후 OPEN·FIST 오분류 체류 <5%, 손 미검출 상황 graceful 처리.
- **잠금·개방**: 손 과제(P5 등)의 **선행 필수**. 마우스 모드는 불필요(숨김).
- **⚠️ 결정 필요**: 15초가 아동 주의 유지에 적절한지(단축·게임화 필요?). 시각 시연(손 애니메이션) 추가 여부.

## P5. 과제 · 물건 옮기기 (StationPhase, move) — 최우선 하드닝
- **목적**: 거울 모드에서 사물을 잡아 목표(드롭존)로 옮긴다.
- **치료 목표(1차)**: **상지 소근육 — 쥐기–놓기 분리 조절 + 능동적 손 폄.** (부수: 눈–손 협응, 도달·조준, 순서화, 지속 주의) · ICF: d440 fine hand use.
- **현재 상태**: 도달→쥐기(OPEN→FIST, R1)→운반→놓기(OPEN 유지·저속, R2)→드롭존 스냅. assist(반경 확대→부유), 추적소실 부유대기, 별점, 커서·게이지.
- **P0 요구사항(사용 가능)**:
  - 아동이 안내만으로 잡기/놓기를 수행. **잡힘/놓임 상태가 즉각·명확**(커서 변화·소리·사물 부착).
  - R1(빈 주먹→다시 펴기)·R2(펴서 유지→놓기)가 아동에게 직관적. 오쥐기·오낙하 낮음.
  - 드롭존 판정 경계가 시각과 일치(존 링).
  - assist가 실패 없이 단계적 발동.
- **화면 요소**: 거울 영상, 사물(3D), 드롭존(존 링), 손 커서(상태색), 안내문, 진행(n/총), (장소 데코·칩).
- **상호작용·상태**: LOCKED→ARMED(OPEN)→CARRY(FIST+대상)→RELEASE(OPEN 유지·저속). 존 밖 놓기=그 자리(실패 아님).
- **수용 기준(=개방)**: 의도 사물 잡기 성공률 ≥80%, 운반 중 오낙하 10회당 ≤1, 놓기가 의도 위치에서 발생, 아동이 튜토리얼 없이 1스테이션 완주.
- **잠금·개방**: 캘리브 선행 필수. 통과 시 개방(1순위).
- **⚠️ 결정 필요**: 잡기 반경·t_release 기본값의 아동 적합성(현장 튜닝). 사물 크기·수 상한(주의 분산 방지).

## P6. 과제 · 걷기 (NavPhase)
- **목적**: 3인칭 항행 구간을 양손 교대 스트로크로 전진, (Lv2) 체간 기울기로 조향.
- **치료 목표(1차)**: **양측 상호교대 협응(reciprocal) + 체간 조절.** (부수: 리듬성 반복, 지속 주의) · ICF: d450 walking.
- **현재 상태**: 교대 스트로크 임펄스 전진, Lv0 자동/1 스트로크/2 조향, 테마별 코스, 아바타 팔 스윙.
- **P0 요구사항(사용 가능)**:
  - 교대 스트로크가 **의도대로만** 전진(오검출로 저절로 나아가지 않음).
  - 아동이 "번갈아 젓기"를 즉시 이해(시각·리듬 큐). Lv2 기울기 방향이 직관과 일치.
  - 진행감·목표(도착 게이트) 명확.
- **화면 요소**: 3D 코스·아바타·도착 게이트, 진행 바, 안내문. (영상 숨김 — 페이즈 교대)
- **상호작용**: 스트로크=전진 임펄스(감쇠), lean=조향(데드존).
- **수용 기준(=개방)**: 교대 스트로크 검출률 ≥90%, 의도 없이 전진 안 함, Lv2 방향 일치, 아동이 구간 완주.
- **잠금·개방**: 캘리브 선행(스트로크는 주먹 판정 사용). 통과 시 개방(2순위).
- **⚠️ 결정 필요**: 걷기의 "실패" 개념 부재 시 도전감 부여 방법(수집물·시간?). Lv2 조향 난이도.

## P7. 과제 · 횡단보도 (CrossingPhase)
- **목적**: 2차선 도로를 신호에 맞춰 안전하게 건넌다.
- **치료 목표(1차)**: **충동·반응 억제(Go/No-Go) + 안전 인식.** (부수: 인과 추론, 신호 감시 주의, 지역사회 이동) · ICF: d460, d240.
- **현재 상태**: 빨간불=연석 전진 잠김/초록불=스트로크로 건넘, 차량·정지선, 신호 음향, Lv0/1.
- **P0 요구사항(사용 가능)**:
  - 신호 상태가 **아동에게 즉각 명확**(색·음향·문구), 빨간불에 확실히 정지.
  - 건너기 조작이 걷기와 일관(스트로크). "언제 가고 언제 멈추는가"가 직관적.
  - 건너는 중 신호 변경 시 차량 대기(위험 종결 없음).
- **화면 요소**: 3D 도로·신호등(적/녹/점멸)·차량·마트 파사드, 진행 바, 상황 안내문.
- **상호작용**: 연석 정지(잠금), 초록불 스트로크 전진, 신호 감시.
- **수용 기준(=개방)**: 빨간불 정지 100%(연석 초과 0), 초록불 완주 가능, 신호 인지 명확, "대기 중 출발시도" 지표로 억제 관찰 가능.
- **잠금·개방**: 걷기 기전 위에 성립 → 걷기 개방 후 권장. 통과 시 개방(3순위).
- **⚠️ 결정 필요**: 좌우 살피기(고개 회전) 추가 여부 — Pose yaw 신뢰성 스파이크 필요(보류). 신호 시간 기본값.

## P8. 과제 · 고르기 (SelectPhase)
- **목적**: 키오스크·버스 노선·길찾기 등 화면에서 정답을 골라 순차 진행.
- **치료 목표(1차)**: **작업기억 + 인과·결과 예측(실행기능).** (부수: 상지 도달·유지, 시각 변별, 선택적 주의, 충동 억제, 지역사회 IADL) · ICF: d210/d179.
- **현재 상태**: OPEN 손 dwell(머무르기)로 카드 선택, 오답 흔들림+기록, 순차 스텝, assist(정답 강조).
- **P0 요구사항(사용 가능)**:
  - dwell 선택이 **손 떨림에 강건**(카드 이탈 유예), 오선택 유발 최소.
  - 카드·안내·스텝 진행이 명확. 무엇을 고르는지 아동이 이해.
  - 오답이 실패로 종결되지 않음(되돌아옴+기록).
- **화면 요소**: 거울 영상, 선택 카드(이모지+라벨), dwell 진행 링, 안내문·스텝, 손 커서.
- **상호작용**: OPEN 손 유지=선택, 오답=흔들림, 정답=다음 스텝.
- **수용 기준(=개방)**: dwell 오작동 없음, 손 흔들림에도 오선택 안 남, 스텝 진행 명확, 아동이 다단계 완료.
- **잠금·개방**: 손 도달·유지 필요(캘리브 선행). 인지 중심 → 후순위 개방(4~5순위).
- **⚠️ 결정 필요**: dwell 시간 기본값(900ms) 아동 적합성. "유지 못 하는 아동" 대체 선택 방식.

## P9. 완료 · 리포트 화면
- **목적**: 아동에게 성취(별)·격려, 치료사에게 수행 관찰 보조(리포트).
- **치료 목표**: 없음(결과·강화). 별 보상=동기, 리포트=관찰 보조.
- **현재 상태**: 총 별·시간, 항목별 요약, JSON/CSV 로컬 다운로드.
- **P0 요구사항**: 아동용 축하와 치료사용 기록을 **역할별로 분리 표기**(아동은 별·격려만, 세부 지표는 치료사 영역). C2 준수 문구.
- **화면 요소**: 축하 제목·총 별, 리포트 박스(항목별 지표), 저장 버튼(JSON/CSV), 온디바이스 안내.
- **상호작용**: 다시 하기 / (개별 연습이면) 메뉴 복귀, 리포트 저장.
- **수용 기준**: 지표가 치료사에게 해석 가능(잡기·오낙하·대기시간·오선택·도움 단계), 다운로드 무오류, 영상·개인정보 미포함.
- **잠금·개방**: 해당 없음.
- **⚠️ 결정 필요**: 아동 화면에 지표 노출 최소화 정도. 회차 간 비교(추세) 필요 여부(현재 세션 단위).

## P10. 치료사 에디터
- **목적**: 코드 없이 아동별 훈련 세션을 저작하고 실행 링크/파일 생성.
- **치료 목표**: 없음(저작 도구). 치료사가 목표별 과제를 조립.
- **현재 상태**: 플로우 편집(이동/횡단보도/옮기기/고르기), 사물 팔레트·드래그 배치, 단계조절, GLB 업로드, 장소, 예산·변별·양손, JSON 저장/열기, 실행 링크.
- **P0 요구사항(Gate 2 대비)**:
  - 미참여 치료사가 무도움 30분 내 스테이션 2개 세션 저작→실행.
  - 각 과제 유형의 목적·조작을 저작 화면에서 이해(간단 설명·미리보기).
  - 저장/불러오기/링크가 안정. 검증 오류 메시지가 명확.
- **화면 요소**: 헤더(제목·저장·링크), 플로우 목록, 배치 캔버스, 사물 팔레트, 속성 패널(단계조절·목표 옵션).
- **상호작용**: 항목 추가·정렬·삭제, 사물 배치, 파라미터 조절, 링크/파일 내보내기.
- **수용 기준(=Gate 2)**: 무도움 30분 내 2스테이션 저작→링크 실행 완료(치료사 1인).
- **잠금·개방**: 러너의 잠금과 별개(치료사 도구). 단, 잠긴 과제 유형을 에디터에서 만들게 할지 ⚠️.
- **⚠️ 결정 필요**: 개방 안 된 과제 유형을 에디터에서 숨길지/경고할지. 저작 온보딩(첫 사용 가이드) 필요 여부.
