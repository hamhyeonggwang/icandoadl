# Problem Brief — ADL 훈련 과제 웹 저작·실행 도구 (웹캠 거울 모드)
날짜: 2026-07-12 · 버전: v0.1 (세션 결정 동결본)
Conformance: problem-brief 스킬(L3) 절차 준수. Designer Contract DC-1~7 대조는 정본(dpd-domain-rules.md·외부 Git) 미확인으로 **Provisional** — 정본 확인 후 Envelope 검증 필요.

## 문제
학령기 작업치료사는 아동별 치료 목표(능동적 손 폄·파악, 양측 상호교대 협응, 체간 자세조절)에 맞춘 디지털 ADL 훈련 과제를 코드 없이 직접 만들 수단이 없어, 목표와 어긋난 기성 콘텐츠를 변통하거나 물리 교구 수작업에 머문다.

## 배경 / 증거
- [미확인·추정] 팀 내 수요 빈도·강도 — 정식 수요 조사 없음. Gate 2 저작 테스트가 간접 검증을 겸한다.
- [사실] 기술 기반 재사용 가능: 기존 제품(퐁퐁 v9+)에서 MediaPipe HandLandmarker(양손·GPU)·getUserMedia 거울 합성·Web Audio 합성음·교대 스트로크 임펄스 물리(Node.js 시뮬레이션 검증) 확보 — borrow-from-self 적용.
- [사실·구조 불변] 병원 데이터 경계: 아동 웹캠 영상·산출물은 전량 온디바이스 처리, 클라우드 업로드 금지.

|---|---|
| 학령기치료팀 치료사 (제작자, v1 사용자) | 코드 없이 30분 내 과제 저작, 아동별 단계조절 |
| 아동 (러너 최종 사용자) | 실패 종결 없는 놀이 경험, 신체 움직임이 곧 조작 |
| 병원 IT·개인정보 담당 | 영상 온디바이스 처리, 저장·전송 없음 |
| 팀장·개발자 (본인) | 단독 유지보수 가능 범위, 기존 자산 재사용률 |

## 확정 설계 결정 (동결)
| 영역 | 결정 |
|---|---|
| 사용자·기기 | 학령기 OT(v1 팀 내부) 저작 → 아동이 웹캠 연결 PC 브라우저에서 링크로 실행, 설치 없음 |
| 입력 문법 | 쥐기: 직전 OPEN 필수(R1) → FIST 전환 ∧ 잡기반경 내. 놓기: OPEN 유지 t_release ∧ 저속(R2). 이동: 양손 FIST 교대 스트로크 임펄스. 조향: 체간(어깨 기준) 측방 기울기 |
| 인식 | canonical 제스처 분류 대신 손 크기 정규화 openness 연속 점수 + 히스테리시스. 세션 시작 15초 개인 캘리브레이션(openness 범위 + 체간 중립) |
| 아키텍처 | 페이즈 교대: 3인칭 항행(패럴랙스, 영상 숨김) ↔ 거울 모드 조작(영상 표시). 카메라 스트림 상시 유지, 렌더링만 토글. 페이즈별 FSM 매핑 교체 |
| 저작 모델 | Session = Station 선형 시퀀스. Station = 사물 배치 + "A를 B에 넣기" 과제 + 성공조건 + 단계조절. Segment(항행)는 파라미터 선택만(Level·길이). 과제는 스테이션 국소 — 인벤토리 없음 |
| 자산·렌더링 | 내장 일상사물 라이브러리 우선 + GLB 업로드. 상호작용은 화면 평면 2.5D, 3D 모델은 렌더링용. 물리엔진 없음(스냅·드롭존 판정) |
| 단계조절 | 이동 Level 0(자동)/1(교대만)/2(교대+조향). 조작: 잡기 반경, 쥐기 허용 손 지정, 조향 데드존, 기울기 상한. 실패호 제거: 타임아웃 → assist(반경 확대 등), 추적 소실 → 부유 대기 |

## 성공기준 (가설 — 검증 가능형)
1. **Gate 1 (제스처 스파이크)**: 대상 아동 2~3명에서 부록 지표(오쥐기율·오낙하율·스트로크 검출률·결합 FPS·기울기 신호 분리) 통과선 충족 — 통과선 수치는 전부 초기 추정.
2. **Gate 2 (저작성)**: 제작 미참여 팀 치료사 1명이 도움 없이 30분 내 스테이션 2개 세션을 저작하고 링크 실행까지 완료.
3. **러너 검증**: 아동 1명이 치료사 동석 하에 세션 1개(스테이션 2개) 완주, 실패 종결 0회(assist 발동은 허용).

## 범위
**Out (하지 않는 것)**
- WebXR 헤드셋·모바일 AR (거울 모드는 Out 아님 — 별도 스택 불요)
- 사진·영상 photogrammetry 복원, 텍스트→3D 생성, USDZ 입출력
- 재질·조명 편집, 타임라인 애니메이션, 물리엔진
- 스테이션 간 운반·인벤토리, 분기형 시퀀스 (A5′·A9)
- 수행 데이터 자동 기록·리포트 (v1, A2 — 치료사 동석 관찰 전제)
- 치료 효과 주장 (비의료기기 경계 유지)
- 개발자용 장면 데이터 export (JSON은 Exit 수단으로만 존재)
- 영상·이미지의 저장·클라우드 전송 일체 (C2 불가침)

**Exit (손을 떼는 방법)**
- 데이터: Session·Station·과제 정의를 JSON(+GLB 참조)으로 저장 — 에디터·러너 분리 구조라 도구 중단 시에도 과제 데이터는 타 러너로 이식 가능.
- Gate 1 실패 시: 손 추적 러너를 접고 입력 드라이버 추상화 위에서 마우스·터치 드라이버로 폴백 — 저작 도구는 그대로 생존.
- Gate 2 실패 시: 치료사 저작을 포기하고 개발자 저작 + 치료사에게 단계조절 파라미터만 노출하는 축소 운영으로 전환.

## 부록 — Gate 1 제스처 어휘 스파이크 스펙 (v0.1)

**목적**: 본 제품 최상위 기술 리스크(아동 대상 손+자세 동시 추적 신뢰성)를 에디터 구축 전에 검증한다.

**형태**: 단일 HTML (퐁퐁 러너 아키텍처 차용). HandLandmarker(numHands:2) + PoseLandmarker(lite, 2~3프레임당 1회 추론). 상태기계 시각 디버그 오버레이(openness 게이지·FSM 상태·기울기 바) 포함.

**측정 항목·통과선 (전부 초기 추정)**
| 지표 | 정의 | 통과선 |
|---|---|---|
| openness 분리도 | 개인 캘리브레이션 후 OPEN·FIST 분포의 히스테리시스 대역(35~65%) 침범률 | 오분류 체류 <5% |
| 오쥐기율 | 의도된 쥐기 시도 중 실패 + 비의도 쥐기 발생 비율 | ≤10% |
| 스윙 중 오낙하 | CARRY 상태 교대 스윙 10회당 낙하 횟수 | ≤1회 |
| 스트로크 검출률 | 관찰된 교대 스트로크 대비 검출 비율 | ≥90% |
| 결합 FPS | Hand(양손)+Pose(lite) 동시 구동 프레임률 | ≥20fps |
| 기울기 신호 | 의도 기울기 시 어깨 변위·롤의 방향 일치율, '의자째 이동' 보상 시나리오와의 구분 가능 여부 | 방향 일치 + 보상 구분 가능 (질적 판정, A8 결정 근거) |
| 보상 패턴 | 치료사 관찰 기록 (주먹 충돌 잡기 시도, 머리 기울기 등) | 질적 |
| 모델 가용성 | 단일 Holistic 계열 웹 지원 여부 문서 확인 | 확인 항목 |

**대상·환경**: 아동 2~3명(비정형 손 자세 ≥1명 포함), 치료실 실제 조명, 웹캠 연결 PC.

**데이터 경계 (C2)**: 영상 프레임의 저장·전송 금지. 랜드마크 파생 수치(openness, 손목 수직속도, 어깨 x변위·롤)만 타임스탬프와 함께 로컬 CSV 다운로드. 아동 식별정보 미포함.
